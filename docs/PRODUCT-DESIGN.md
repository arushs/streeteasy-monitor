## Executive Summary

Transform StreetEasy Monitor from a single-user listing tracker into a multi-user platform with automated landlord/broker contact capabilities. Users forward StreetEasy alerts to the system, which extracts listings and can auto-contact on their behalf.

**Core Value Proposition:** "Never miss a listing. Never type another inquiry email."

## 7. Design Recommendations

### 7.1 Critical UX Decisions

1. **Email Forwarding Over OAuth**
   - Simpler setup (just forward emails)
   - Works with any email provider
   - No scary "access your email" permission
   - Unique address per user = natural multi-tenancy

2. **Review-First Default for Auto-Contact**
   - Reduces spam anxiety
   - Lets users learn the system
   - Upgrade path to full auto for power users

3. **Mobile-First Design**
   - NYC renters check listings on the go
   - Quick swipe actions (save/skip/contact)
   - Push notifications for responses

4. **Batch Operations**
   - Contact multiple listings at once
   - Review queue before sending
   - Daily limits to prevent spam flags

### 7.2 Trust & Safety UX

1. **Transparent Limits**
   - Show daily contact count (X/10 today)
   - Explain why (spam prevention)
   
2. **Easy Opt-Out**
   - One-click stop auto-contact
   - Clear "pause" vs "delete account"

3. **Data Visibility**
   - Users see exactly what's sent
   - Full contact history accessible
   - Export your data anytime

### 7.3 Technical Recommendations

1. **Use Clerk or Auth0** for authentication
   - OAuth ready (Google, Apple)
   - Handles email verification
   - Nice UI components

2. **Email Processing Pipeline**
   - Unique address per user: `{userId}+se@listings.app`
   - Parse with Mailgun/SendGrid incoming hooks
   - Store raw + extracted data

3. **Template Engine**
   - Mustache/Handlebars for variables
   - Preview renders with real data
   - Validate all variables before send

4. **Response Tracking**
   - Match reply-to addresses
   - Or use Mailgun engagement tracking
   - Webhook on reply for notifications
